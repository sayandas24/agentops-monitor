## Color Scheme

### LLM Calls
- Background: Blue to Indigo gradient (`from-blue-50 to-indigo-50`)
- Border: Blue (`border-blue-200`)
- Text: Blue tones for emphasis
- Input tokens: Green
- Output tokens: Blue

### Tool Calls (Success)
- Background: Green to Emerald gradient (`from-green-50 to-emerald-50`)
- Border: Green (`border-green-200`)
- Badge: Green background

### Tool Calls (Failed)
- Background: Red to Pink gradient (`from-red-50 to-pink-50`)
- Border: Red (`border-red-200`)
- Badge: Red/destructive variant
- Error box: Red background with red text

### Statistics Cards
- Execution: Purple to Pink gradient
- LLM Stats: Blue to Indigo gradient
- Tool Stats: Green to Emerald gradient
- Cost: Yellow to Orange gradient

## Key Features Highlighted

### 1. Progressive Disclosure
- Expandable sections keep the UI clean
- Click to reveal detailed information
- Character counts help users decide what to expand

### 2. Visual Hierarchy
- Icons (ğŸ¤–, ğŸ”§, âš™ï¸) for quick identification
- Section headers with counts
- Color-coded cards by type
- Clear separation between sections

### 3. Metrics at a Glance
- Summary cards at the top
- Key metrics prominently displayed
- Ratios and calculations shown
- Free tier indicators

### 4. Error Visibility
- Failed operations clearly marked
- Error messages prominently displayed
- Red color scheme for failures
- Error counts in statistics

### 5. Cost Transparency
- Cost shown at multiple levels
- Per-call and per-token breakdowns
- Free tier badges
- Zero-cost operations highlighted

## Responsive Design

### Desktop (1200px+)
- 4 columns for summary cards
- 3 columns for trace cards
- Side-by-side layouts for details

### Tablet (768px - 1199px)
- 2 columns for summary cards
- 2 columns for trace cards
- Stacked layouts for some sections

### Mobile (< 768px)
- Single column layout
- Stacked cards
- Collapsible sections more important
- Touch-friendly tap targets

## Accessibility Features

- Semantic HTML structure
- Proper heading hierarchy
- Keyboard navigation support
- ARIA labels where needed
- Sufficient color contrast
- Focus indicators
- Screen reader friendly

## Performance Considerations

- Lazy loading of detailed data
- Expandable sections reduce initial render
- Efficient filtering and mapping
- Memoization opportunities
- No unnecessary re-renders
